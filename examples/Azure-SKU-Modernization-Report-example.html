Report guide - concepts, fields and sections

Core concepts

Compute retirement path
Standalone VMs plus Public Preview Batch/VMSS sidecars whose VM size has a retirement signal. Remediation waves still cover standalone VMs only.
Advisor-confirmed
Per-resource Advisor signal. It is stronger than a generic family exposure because Azure identified the specific VM.
SKU-family exposure
The current VM size belongs to a retiring SKU family from Microsoft Learn, even when Advisor did not emit a per-resource row.
Monitoring lifecycle
Dependency Agent / VM Insights Map lifecycle findings. These are intentionally separate and never counted as compute SKU retirements.

Wave model

W0 Time-critical
Retirement risk is Critical/High. Calendar pressure wins over all other sequencing rules.
W1 Advisor + sensitive
Advisor-confirmed VM on a sensitive workload. Use tighter change governance and validation.
W2 Sensitive validation
Sensitive workload with same-generation resize. Lower technical change, but still validate workload behavior.
W3 Architecture
Cross-family Gen1→Gen2 path. Validate image, drivers, boot/runtime assumptions, capacity and performance profile.
W4 Low complexity
Standard same-generation resize lane. Best candidate for runbook-driven batch remediation.

Decision sections

Decision Room
Management view: what to act on now, what needs validation, and what can move as quick wins.
Risk vs Effort Matrix
Translates waves into execution lanes so risk and engineering effort are visible at a glance.
Execution Scenarios
Conservative, balanced and accelerated views over the same facts. They do not change counts or recommendations.
If We Do Nothing
Earliest dated retirement rows, useful as the escalation queue for planning.

Detail table fields

Recommended SKU
Deterministic candidate target selected by the script; validate compatibility, quota, capacity and business constraints before change.
Retail cost delta / month
PAYG/list-price estimate only. It excludes real negotiated pricing, RI/Savings Plan effects and workload-specific commercial adjustments.
Validation
Checks and caveats that should be reviewed before migration, including generation change, sensitive workload and RI/SP flags.
Next step
Suggested operational action for the row. It is guidance, not an approval to migrate.

Analysis Coverage

What this script covers: Azure Advisor recommendations in the Reliability → Service Upgrade and Retirement subcategory, across all SKU families (no fixed list), with retirement date / retiring feature when available.

What it does NOT cover / manual verification required:

  • Retirements present only in Azure Service Health (not emitted by Advisor). → Check Service Health → Health advisories and the Impacted Resources tab.
  • Services not yet covered by the Service Retirement Workbook (partial coverage). → Verify in Advisor → Workbooks → Service Retirement.
  • Public announcements without a mapping to a resource. → Check Azure Updates.
  • Service Health retention in Azure Resource Graph is 90 days: reconcile within that window.
